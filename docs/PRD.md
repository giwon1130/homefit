# PRD — homefit

## 1. 비전

"내 정보 한 번 넣으면 → **넣을 수 있는** 청약만, **출퇴근 가능한** 지역만, **감당되는** 금액만 추려서 보여주는 앱."

## 2. 타겟 유저

### 페르소나 A — 30대 신혼부부 (맞벌이)
- 혼인 3년차, 자녀 없음, 부부합산 연소득 9,000만 ~ 1.3억
- 둘 다 서울 강남/종로 직장, 전세 거주
- 청약 제도가 복잡해서 어디부터 봐야 할지 모름
- 니즈: "우리가 넣을 수 있는 청약 + 통근 되는 곳 + 우리 예산"

### 페르소나 B — 30대 후반 자녀 있는 가족
- 자녀 1~2명(미취학), 무주택, 세대주
- 외벌이 또는 맞벌이, 연소득 7,000만 ~ 1.1억
- 전세 만기 압박, 다자녀 특별공급 존재 여부조차 모름
- 니즈: "다자녀/신혼부부/생애최초 중 어느 카드가 유리한지"

## 3. 유저 스토리 (MVP)

### US-1 온보딩
> As a 유저, 카카오/구글로 로그인하고 프로필을 입력하면, 내 청약 가점과 유형별 자격이 자동으로 계산되어 보이기 원한다.

### US-2 맞춤 청약 목록
> As a 유저, 내가 넣을 수 있는 청약만 필터링된 목록을 보고, "내 조건 적합도" 순으로 정렬되기 원한다.

### US-3 통근 가능성
> As a 유저, 각 청약의 상세 화면에서 본인과 배우자의 실제 도어투도어 통근시간(환승 포함)을 보고 싶다.

### US-4 유형 비교
> As a 유저, 한 청약에 대해 "신혼부부 특별공급" vs "생애최초" 중 어느 쪽으로 넣는 게 당첨확률이 높은지 비교하는 카드를 보고 싶다.

### US-5 알림
> As a 유저, 관심 청약의 접수 1일 전/당일 아침/마감 2시간 전에 푸시 알림을 받고 싶다.

### US-6 캘린더
> As a 유저, 월/주 단위로 접수 일정을 한눈에 보고 싶다.

### US-7 대안 추천
> As a 유저, 관심 지역의 청약이 너무 경쟁률 높거나 당첨이 어려우면, 근처 전세/월세/매매 대안을 같이 보고 싶다.

### US-8 단지 정보
> As a 유저, 상세 화면에서 단지 평면도/조감도/입지 사진을 보고 싶다.

## 4. 기능 요구사항 (MVP → 확장)

### 4.1 프로필
- 기본: 이름, 생년월일, 혼인일, 세대구성(자녀 수/생년)
- 자산: 부부합산 연소득(근로/사업 구분), 순자산, 청약통장(가입일/납입횟수/납입금액)
- 주택 이력: 무주택 기간, 과거 소유 여부/처분일
- 지역: 현 거주지(주소+거주기간), 회사 주소 최대 2개(본인+배우자)
- 선호: 예산(매매/전세/월세 각각), 최소 방 개수, 최대 통근시간, 선호 지역

### 4.2 청약 탐색
- 통합 데이터: 민영분양, 공공분양, 신혼희망타운, 행복주택, 매입임대, 전세임대, 국민임대
- 필터: 지역(시/도/구), 가격 범위, 전용면적, 접수기간, 공급 유형
- 정렬: 내 매칭 스코어 / 접수 임박순 / 가격순

### 4.3 자격 판정 엔진
- 입력: 프로필 + 청약 공고의 규칙(rule_json)
- 출력:
  - 신청 가능 유형 배열 (예: `["생애최초", "신혼부부", "일반"]`)
  - 각 유형별 예상 경쟁률 밴드 (상/중/하)
  - 추천 유형 및 근거 텍스트
- 룰 엔진: 소득기준표, 자산기준, 무주택 요건, 청약통장 요건, 지역 거주 요건 등

### 4.4 통근 계산
- 입력: 단지 위치, 프로필의 직장 좌표들, 출근 시간대(기본 평일 08:00 도착)
- 처리:
  - 1차: PostGIS `ST_DWithin` 로 10km 이상 떨어진 것 제외
  - 2차: ODsay/카카오모빌리티 API 호출, 도보+대중교통 최단 경로
  - 결과 캐싱 (같은 출발/도착은 30일 캐시)
- 출력: 본인/배우자 각각의 분, 환승 횟수, 경로 요약

### 4.5 매칭 스코어
- 구성 요소 (가중치는 튜닝 대상):
  - 자격 적합도 30%
  - 통근 점수 30% (본인/배우자 평균, 최대값 허용치 반영)
  - 예산 적합도 25%
  - 지역 선호 10%
  - 세대 적합도(방/면적) 5%
- 0~100 점수로 산출

### 4.6 알림
- 종류: D-1, 접수 당일 09시, 마감 2시간 전, 당첨 발표일
- 설정: 관심 청약(즐겨찾기) 대상만, 혹은 스코어 70 이상 자동
- 전송: APNs(iOS) / FCM(Android) / 웹 푸시

### 4.7 캘린더
- 월/주 뷰, 접수 시작일 하이라이트
- `.ics` 내보내기 (iOS/Google Calendar 연동)

### 4.8 대안 매물
- 트리거: 청약 스코어 60 미만이거나 경쟁률 밴드 "상"
- 소스: 국토부 실거래가 API (최근 3개월)
- 필터: 같은 시/구, 예산 ±20% 이내, 면적 유사

### 4.9 지도
- 단지 위치, 직장과의 경로 미리보기
- 카카오맵 SDK (웹: JS SDK, 모바일: RN 웹뷰 또는 네이티브 모듈)

### 4.10 단지 상세
- 공고 원본 링크, 평면도 이미지, 조감도, 입지 사진
- 내 자격 카드, 통근 카드, 실거래 비교 카드

## 5. 비기능 요구사항

| 항목 | 목표 |
|---|---|
| 응답속도 (청약 목록 p95) | < 500ms |
| 응답속도 (상세) | < 800ms |
| 통근 API 캐시 적중률 | > 80% |
| 개인정보 저장 | AES-256 필드 암호화 (소득/자산/주소) |
| 접근성 | 다이나믹 타입, VoiceOver/TalkBack, 한국어 우선 |
| 크롤링 | robots.txt 준수, Rate 1 req/sec 이하 |

## 6. 비범위 (MVP에서 제외)

- 분양권 전매/재건축 예측
- 모기지/대출 한도 계산 (Phase 4 이후)
- 커뮤니티/리뷰
- 안드로이드 (Phase 3 이후)
- 중개사 연결 / 거래 기능

## 7. 성공 지표 (North Star 후보)

- **MAU 대비 "자격 판정 카드 확인" 비율** (제품의 고유 가치 지표)
- 알림 클릭률 (D-1 푸시 기준)
- 즐겨찾기 → 실제 청약 접수 전환율 (정성 리서치로 측정)
